About Maestro Server

Open source software platform for management and discovery of servers, applications and systems for Hybrid IT.

What problems does it solve?

Maestro was built to solve some problems faced in the operation of multicloud environments, multi-shared devops culture and multiple clients, where it becomes hard to (a) keep track of the lastest environment state, (b) apply a compliance in all teams, (c) accurately visualize your infrastructure state, (d) manage access for internals employees and (e) maintain the documentation up-to-date.

  1. How can we audit our environments?
  2. How can we control and keep track of our environment?
  3. How can we guarantee if our documentation is up-to-date?
  4. Which servers belong to a given client?

What Maestro does?

Maestro comes to (a) help IT operation teams to organize and audit multicloud infrastructure; (b) substitute CMDB systems; (c) enable continuous discovery of servers, services and apps; (d) classify each service as a database, message queue, VPN, API gateway, service mesh, etc., and; (e) create a relation between servers, services and clients.

Maestro comes for you, to be a complete and simple cloud inventory.

How do I use it?

Analyse your full state environment of all providers do you have, centralize all information about datacenters, servers, loadbalance, orchestrations tools, volumes, vpns and etc.


  • - Keep track their relations
  • - Create complex and powerful reports
  • - Analysis costs and growing up velocity
  • - Standards services names
  • - Network configurations
  • - Available deploys for each server
